Raspberry PI Server & PIC32MM

Embedded systems projects: portable server and microcontroller programming

Raspberry Pi Zero W Portable Server

Configured a Raspberry Pi Zero W to act as a compact, portable server with significant storage capacity. Despite the board's tiny footprint, the system provides network-accessible storage and demonstrates the versatility of Linux-based embedded computing.

System Configuration

  • Platform: Raspberry Pi Zero W with WiFi connectivity
  • Operating System: Raspberry Pi OS (Linux-based)
  • Storage: 4TB external storage via USB adapters
  • Network: WiFi-based file server accessible over local network
  • Services: Samba/NFS for network file sharing

Key Features

  • Portable network-attached storage (NAS) solution
  • Remote access via SSH for administration
  • Low power consumption for always-on operation
  • File synchronization and backup capabilities
  • Media server for streaming content

Use Cases

This portable server serves as a personal cloud storage solution, providing wireless access to files from any device on the network. The compact form factor makes it ideal for travel, backup purposes, and as a development platform for testing server applications.

PIC32MM Curiosity Development Board

Programming and experimentation with the PIC32MM microcontroller, focusing on fundamental embedded concepts including GPIO control, interrupts, and hardware timers. This project demonstrates low-level firmware development and hardware interaction.

Implementation Details

  • Platform: Microchip PIC32MM Curiosity Development Board
  • Programming: C language for embedded firmware
  • Development Tools: MPLAB X IDE, XC32 compiler
  • Features: 4 LED states controlled by switches and timer
  • Hardware: Integrated 555 timer circuit for timing operations

Technical Components

  • GPIO configuration for LED and switch control
  • Interrupt service routines (ISRs) for switch inputs
  • Timer configuration using 555 timer integration
  • State machine implementation for LED patterns
  • Debouncing logic for reliable switch detection

Learning Outcomes

This project provided hands-on experience with microcontroller programming, interrupt-driven design, and hardware-software integration. Working with the PIC32MM reinforced concepts in embedded systems development, register-level programming, and real-time constraints.

Embedded Systems Concepts

Both projects demonstrate different aspects of embedded computing: the Raspberry Pi showcases Linux-based embedded systems with networking capabilities, while the PIC32MM represents bare-metal microcontroller programming with direct hardware control.

These complementary experiences provide a broad foundation in embedded development, from high-level operating systems to low-level firmware, essential for modern hardware-software co-design work.
