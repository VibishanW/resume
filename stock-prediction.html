Hardware Accelerated Stock Prediction (LSTM RNN)

A full-stack project combining Python model development and FPGA deployment on AMD/Xilinx U280 to predict next-day S&P 500 values (Open, Close, High, Low, Volume).

Project Overview

This project starts with a Python LSTM-RNN model trained on 2020 SPY market data, then extends into hardware-accelerated inference on a Xilinx U280 using Vitis HLS and XRT. The repository includes software baselines, a conventional RNN hardware step, a full LSTM hardware implementation, and analytics scripts for accuracy/runtime comparison.

Repository: Stock_Prediction_Via_LSTM_RNN

Software Model (Python)

  • Uses yfinance to download SPY daily candles from 2020-01-01 to 2021-01-01.
  • Input features: Open, Close, High, Low, Volume.
  • Per-feature MinMax normalization and a 60-day sequence window.
  • Model architecture: 2 stacked LSTM layers (50 units each) and a 5-output dense head.
  • Predicts next-day 5-feature vector; exports plots and text outputs for analysis.

Hardware Path (U280)

  • HLS kernel implementation in C++ for LSTM/RNN paths with Vitis 2023.2 flow.
  • Host-side runtime uses XRT to load xclbin, stream inputs, execute kernels, and collect outputs.
  • LSTM design includes gate computations with fixed-point math and clipping for numerical stability.
  • Supports multi-file input batches and output consolidation for software-vs-hardware analytics.

Implementation Notes & Engineering Analysis

  • The software path uses a straightforward and reproducible workflow (data ingest → normalize → train → infer), making it a strong baseline for hardware comparison.
  • Hardware flow introduces additional engineering constraints: fixed-point considerations, host-device transfer overhead, and synthesis/tool constraints.
  • The project is strong in methodology: it uses the same signal set and direct output-file comparisons to produce transparent accuracy metrics.
  • The split between incremental milestones (RNN first, then LSTM) demonstrates practical risk reduction for FPGA algorithm bring-up.

Future evolution could include larger rolling windows, richer feature engineering (e.g., volatility indicators), and quantization-aware training to better align software training behavior with fixed-point hardware execution.
