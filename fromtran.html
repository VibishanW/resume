From the Transistor to the Web Browser

A comprehensive journey through computer engineering, inspired by George Hotz's radical approach to learning

A project idea developed by George Hotz. This ambitious undertaking follows Hotz's vision of understanding computer systems from first principles - literally starting with transistors and building up to a complete web browser.

My engineering mentality is heavily influenced by an avant-garde approach to learning computer science outlined by George Hotz in a document labeled "From the Transistor to the Web Browser". In this document, Hotz tears down flaws in modern computer science curriculum and introduces a more radical approach.

Below you'll find my evolving notes and section-by-section progress through this journey. Each section represents a fundamental layer of computing, built upon the previous foundations.

02

What Language is Hardware Coded In?

Objectives

  • Test an FPGA by blinking an LED
  • Build a simple UART communication interface
  • Get hands-on experience with Verilog/VHDL

This section was meant to get my feet wet with hardware description languages (Verilog/VHDL). With my current professional experience in FPGA development, I decided to move straight to Section 3 for now. I plan to circle back to this foundational exercise to document the learning process for others.

Planned - Will Revisit
03

Processor: What is a Processor Anyway?

Building Blocks

  • An assembler in Python
  • A simple Single-Cycle RISC-V processor in VHDL
  • A bootrom for initial program loading

In this section, I dive deep into processor architecture by implementing a complete RISC-V processor from scratch. This involves understanding instruction sets, designing the datapath and control logic, and creating the toolchain necessary to program it.

In Progress
04

Compiler

Building the Software Stack

  • A C compiler and parser in Haskell
  • A linker in Python
  • Libc + malloc implementation in C
  • An ethernet controller
  • A bootloader

This section bridges the gap between hardware and high-level programming. By building a complete C compiler toolchain, I'll understand how source code transforms into machine instructions that run on the processor built in Section 3.

Planned
05

Operating System

System Software Components

  • Build an MMU (Memory Management Unit) in VHDL
  • Build an Operating System in C
  • Add SD card interface
  • Implement FAT Filesystem in C
  • Add Shell functionality in C

The operating system layer provides abstraction between hardware and applications. This section involves creating memory management, file systems, and a user interface - the essential components that make a computer usable.

Planned
06

Browser

Networking and Applications

  • TCP stack implementation in C
  • telnetd for multiprocessing in C
  • Dynamic linking in C
  • Web browser in C

The culmination of the software stack - implementing network protocols and building a web browser. This demonstrates how modern applications leverage all the layers built in previous sections, from transistors to high-level networking.

Planned
07

Physical Implementation

Hardware Realization

  • Add USB MCU to JTAG functionality in C
  • Build and test everything on actual hardware
  • Optional functionality and optimizations

The final section brings everything into the physical world. Taking the complete system from simulation to actual silicon, testing on real FPGA hardware, and validating that every layer works together as designed.
